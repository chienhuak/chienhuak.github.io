請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

我曾經在公司擔任約聘。
當時我獨自負責許多繁瑣的文書工作，所以我投入了大量時間和精力來將其自動化。
然而，公司後來決定讓其他正職員工接手這項工作，令我感到非常失落。
儘管如此，秉持著團隊精神，我也將這些自動化工具分享給同事。

然而，這並不是結束...
我卻發現有些同事只願意做最基本、已經自動化的任務，而手動工作卻推給我。
更讓我感到挫折的是，雖然我重複教學很多次，同事還在背後說我沒有教她所以他才不會做。

在這段經歷，我選擇去思考問題背後的問題。
並主動尋找解決方案，例如：當之後又遇到要做教育訓練的時候 :
我寫了SOP、錄製教學影片、指派作業、討論職責範圍、Q&A..等等，以幫助新同事快速上手。
這不僅可以保護我自己，還能夠幫助其他人更快適應工作。

這段經歷讓我學會了更加有效地溝通和協作。
也認知良好的團隊文化才能夠讓團隊正向成長。
即使遇到不好相處的團隊成員，我也願意不斷努力去建立和維護這些關係，並從中獲得成長和收穫。
我相信這些經驗將會使我成為一名更好的團隊成員，並且更好地應對未來的挑戰。

關於這份申請網頁，分享一個開發時的技術心得。

我想分享我在個人網頁呈現作品的歷程。

首先，我想用表格列出幾項 Side projects ，及相關超連結。
一開始直覺想要用 table 做，但又覺得修改上很不方便。
後來我找到 iframe 可以將 googlesheet 內容匯到網頁。
這個方法雖然有用，但是樣式很醜，也嘗試了使用 CSS 變更樣式。
另外也嘗試使用 iframe 將 google docs, google slides 連結到網頁。
一番研究卡關後，最後才呈現大家看到的樣子。

然後，做了上述的文字介紹後，我也想把作品放上Github。
這邊我又遇到一個困難，Github似乎無法理解Excel裡的VBA。
在我還不知道git的時候，我的版本控管超級亂。


可以想像以EXCEL來說，就更可怕了。
除了程式變動外，還有分頁內容的變動。
因此，我研究了一些第三方的工具，希望同樣用git控管VBA版本。
例如 rubberduck, XVBA, xwings, xltrail, git XL...等等。
但是這些工具使用方法需要學習，不是那麼直覺，而且有些需要付費。

最後，經過一番折騰。
我竟然是用最簡單的方法，從VBA裡一一複製出.bas檔案，再上傳到 Github。
我想這可能不是最好的方法，如果我有研究出更好的點子，再分享出來。
